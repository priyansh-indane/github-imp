1. git clone < https link >  (clones the GitHub repo into local system).



2. Cd --> change directory.



3. ls -->list files in the directory.



4. git status  (checks the status for repository , checks if there are any untracked or modified files).


5. git add <file-name> (adds new or changed files your working directory to the git staging area).



6. git add . (add all the files to working directory).



7. git commit -m "some message" (It is the record of the change).



8. git push origin main (push-upload local repo content to remote repo. Adds local code from system to github).



9. cd .. (come back to previous directory) .



BASIC WORKFLOW:-

GITHUB REPO-->CLONE-->CHANGES-->ADD-->COMMIT-->PUSH
